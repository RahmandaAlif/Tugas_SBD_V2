Tahapan Merancang ERD

  1. Menentukan entitas
  2. Menentukan Atribut yang termasuk atribut kunci
  3. Identifikasi relasi
  4. Menentukan kardinalitas

Studi kasus : ERD Tentang Penjualan Barang Online

ERD Penjual merupakan aktifitas menjual produk, bisa berupa barang ataupun jasa. Aktifitas ini di lakukan oleh dua pihak yakni penjual dan pembeli. Pada studi kasus kali ini kita akan coba membuat perancangan basis data menggunakan pemodelan
Aturan
  1. Seorang penjual dapat menjual banyak barang
  2. Seorang pembeli dapat membeli lebih dari satu barang yang dijual penjual
  3. Stok barang akan berkurang sesuai jumlah barang yang dibeli
  4. Penjual, pembeli, dan barang dapat diidentifikasi dengan id yang berbeda(unik)

Tahapan 1 : Menentukan Entitas

  • penjual, barang, pembeli

Tahapan 2 : Menentukan Atribut

    Penjual

  • Id_Penjual : integer(pk)
  • Nama_Penjual : string
  • Email : string
  • No_telp : string

    Barang

  • Id_Pembeli : integer(pk)
  • Nama_Barang : string
  • Harga : integer
  • Stok : integer

    Pembeli

  • Id_Pembeli : integer(pk)
  • Nama_Pembeli : string
  • Alamat : integer
  • Email : integer

Tahapan 3 : Menentukan Relasi dan Kardinalitas

  • Kardinalitas relasi antara penjual dan barang adalah one to many karena seorang penjual dapat menjual banyak barang atau banyak barang dapat dijual oleh seorang penjual
  • Kardinalitas relasi antara pembeli dan barang adalah one to many karena seorang pembeli dapat membeli banyak barang atau dengan kata lain banyak barang boleh dibeli oleh seorang pembeli

    Latihan 2

    Rancanglah diagram E-R dari kasus aplikasi database sederhana untuk sistem informasi akademis suatu universitas.

    1. Entitas yang ada
    • Mahasiswa
    • Dosen
    • Mata_kuliah
    • Ruang
    2. Menentukan atribut masing-masing entitas dan menentukan Primary key-nya
    1. Mahasiswa memiliki atribut
      • NRP : Nomor Registrasi Pokok (integer) sebagai Primary key
      • Nama_Mhs : Nama lengkap Mahasiswa (string)
      • Alamat_mhs : Alamat lengkap mahasiswa (string)
    2. Dosen Memiliki atribut
      • NIP : Nomor induk Pegawai (integer) sebagai Primary key
      • Nama_dsn : nama lengkap dosen (string)
      • Alamat_dsn : Alamat lengkap dosen (string)
    3. Mata_kuliah memiliki atribut
      • Kode_MK : Kode untuk mata kuliah (string) sebagai Primary key
      • Nama_MK : Nama lengkap mata kuliah (string)
      • Deskripsi : Deskripsi singkat mengenai mata kuliah (string)
    4. Ruang memiliki atribut
      • Kode_Ruang : kode untuk ruang kelas (string) sebagai Primary key
      • Lokasi : lokasi lengkap ruang kelas (String)
      • Kapasitas : jumlah mahasiswa yang dapat ditampung pada satu ruangan keas (integer)
    3. Menentukan Kardinalitas relasi
    1. Ruang dijadwalkan untuk mata_kuliah
      • Tabel utama : Ruang
      • Tabel Kedua : mata_Kuliah
      • Relasi : One to One (1-1)
    2. Dosen mengajar mata_kuliah
      • Tabel utama : Dosen
      • Tabel Kedua : mata_Kuliah
      • Relasi : One to Many (1-M)
    3. Dosen membimbing mahasiswa
      • Tabel Utama : Dosen
      • Tabel Kedua : Mahasiswa
      • Relasi : One to Many (1-M)
    4. Mahasiswa mengambil mata_kuliah
      • Tabel Utama : Mahasiswa, Mata_kuliah
      • Tabel Kedua : mhs_mengambil_mk
      • Relasi : Many to Many
    4. Membuat ERD secara lengkap
